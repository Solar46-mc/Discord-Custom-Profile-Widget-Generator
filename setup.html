Discord側の下準備

Widgets V2 はまだ一般公開UIがない実験的機能です。コードを生成する前に、Developer Portal側でアプリケーションと ウィジェットのレイアウトを作成しておく必要があります。ここで作った「Data Field」の名前は、作成ページが 表示するキー名（例: display_name）と一致させてください。

1. アプリケーションを作成し、Social SDK を有効化する

Discord Developer Portal で新規アプリケーションを作成し、 サイドバーの Games → Social SDK から申請フォームを送信してください。ゲーム本体は不要で、送信するとすぐにアクセスできます。無料で利用できます。

2. ウィジェット編集画面を解放する（実験フラグ）

Widget エディタは既定では非表示です。Developer Portal を開いた状態でブラウザの開発者コンソールに次のコードを貼り付けて実行すると、 サイドバーに Widget 項目が現れます（ページはリロードしないこと。リロードすると解放状態は消えます）。

let _m=webpackChunkdiscord_developers.push([[Symbol()],{},r=>r.c]);
webpackChunkdiscord_developers.pop();
let f=(...p)=>{for(let m of Object.values(_m))try{
  if(!m.exports||m.exports===window)continue;
  if(p.every(x=>m.exports?.[x]))return m.exports;
  for(let e in m.exports)
    if(p.every(x=>m.exports?.[e]?.[x]) && m.exports[e][Symbol.toStringTag]!=='IntlMessagesProxy')
      return m.exports[e];
}catch{}};
f("getAll").getAll().find(e=>e.getName()==="ApexExperimentStore")
  .createOverride("2026-03-widget-config-editor", 1);
3. ウィジェットのレイアウトを、作成ページと同じ構成で作る

Widget Top（Hero / Contained）、Widget Bottom（Stats Grid / Progress / Collection）、Mini Profile （名前＋説明 / 最初の統計 / カスタムテキスト）を、作成ページで選んだものと同じ組み合わせで作成してください。 各項目の Value Type = User Data を選び、Data Fieldに作成ページが表示するキー名 （例: display_name, stat1_value）をそのまま入力します。設定後は Save Changes → Publish を忘れずに。

4. App ID / Bot Token / あなたのDiscordユーザーIDを控える

App ID は General Information タブ、Bot Token は Bot タブの Reset Token から取得できます。 自分のプロフィール用に使う場合は、Discord の設定 → Authorized Apps で自分のアプリを認可しておいてください （フレンドリスト管理・ステータス更新・プロフィールアクセスの権限が必要です）。 これらの値は作成ページのフォームに入力しますが、ページの外へは送信されません。

5. プロフィールへウィジェットを表示する

作成ページで生成したコードを実行するとデータの送信（PATCH）はできるようになりますが、ウィジェット自体をプロフィールに表示するには、 Discord公式クライアントの「Add Widget」メニューから自分のアプリケーションを追加する操作が別途必要です （実験 2026-03-application-widget-v2-renderer を Variant 1 にしておくことも忘れずに）。 この操作手順はDiscordの仕様変更が頻繁なため、最新のやり方は Discord Previews コミュニティ等で確認することを推奨します。
