05

実行と定期更新

Node.js の場合

Node.js 18 以上であれば追加ライブラリなしで動きます。ファイルを保存して次のコマンドで実行してください。

node widget-updater.js

常駐させたい場合は pm2 start widget-updater.js --name discord-widget のようなプロセスマネージャの利用がおすすめです。 Spotify / Steam / Last.fm 連携は内部で一定間隔ごとにポーリングし続けます。手動入力（画像・統計・コレクション・進捗）は一回だけ送信するので、 値を変えるたびに再実行するか、cronやタスクスケジューラで定期実行してください。

Google Apps Script の場合

script.google.com で新規プロジェクトを作成し、生成されたコードを貼り付けます。 一度 pushToDiscord を手動実行して動作確認し、問題なければ左メニューの時計アイコン（トリガー）から 「時間主導型」トリガーを追加し、数分おきに pushToDiscord を呼ぶよう設定してください。

プロフィールへウィジェットを表示する

まだDiscord側の準備が済んでいない場合は Discord側の準備ページ を先に確認してください。 ウィジェット自体をプロフィールに表示するには、Discord公式クライアントの「Add Widget」メニューから自分のアプリケーションを 追加する操作が別途必要です（実験 2026-03-application-widget-v2-renderer を Variant 1 にしておくことも忘れずに）。
